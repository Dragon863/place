Place - API Docs!

  • On the web demo...
  • Or on my blog.

Base URL

The base URL for the API is https://place.danieldb.uk. All routes are prefixed with this URL.

Endpoints

> GET /api/get_state

This route gets the canvas' current state. It does not accept any arguments. It will return a 2D array, with each pixel represented as an array of the R, G and B values of the pixel. Example response:

            [
    [
        [
            0,
            0,
            0
        ],
        ...
    ],
    ...
]

> GET /api/get_individual_pixel

This route gets the RGB values of a single pixel. It accepts two arguments as query parameters, x and y, which are the coordinates of the pixel. Example request: /api/get_individual_pixel?x=0&y=0. Example response:

            [
    255,
    0,
    0
]

> GET /api/get_counter

This route gets the current value of how many pixels have been placed or updated on the canvas since logging began. It will return with the value only. Example response:

            42

> POST /api/set_pixel_color

This route sets the color of a single pixel. It accepts five arguments as JSON body: x, y, r, g and b (X coordinate and Y coordinate on the canvas, red, green and blue intensity out of 255). Example responses:

            {"success": True} or {"success": False, "error": "Invalid coordinates"}

A note on Bots

Unpopular opinion: I don’t think bot protection needs to be there for a project like this. I think it’s interesting to see what bots can create, and it’s a fun challenge to try and outsmart them. I’ve seen some really cool bots from running this project for a while, and I think it adds to the experience. That being said, I do have some basic rate limiting in place to prevent abuse, but I think it’s important to strike a balance between protecting the project and just seeing people have fun reverse engineering it and creating cool stuff!

This is a quote from my post about this project on my blog. I beleive that a fair use of bots can be beneficial to the project, and I encourage people to experiment with them. However, I do have some basic rate limiting in place to prevent abuse, and if bots are abused to spread hate speech or other harmful content, I will take action to prevent that. I want to keep this project a fun and creative space for everyone!
